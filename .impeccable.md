# Impeccable Context — Deepwork Mind

## Design Context

### Users
Developers and small teams whose coding agents (Claude Code, OpenCode, Cursor) currently start from scratch every session. Later: enterprise orgs who need to find patterns in how work gets done at scale. The user opens the dashboard after `docker compose up` and should immediately feel their setup just got smarter.

### Brand Personality
**Calm intelligence.** Three words: *quiet, cumulative, effortless.*

This is not a tool that demands attention or action. It's a mind that works in the background — when you check in, it's already learned things. The feeling on first open should be: "oh, my harness just got overpowered with a good mind." Not "here's a dashboard full of things to configure."

### Aesthetic Direction

**Reference**: Wispr Flow (wisprflow.ai) — warm editorial feel, generous whitespace, typographic hierarchy through restraint, "quiet luxury" palette. Less SaaS dashboard, more editorial Sunday.

**Theme**: Light. Warm neutrals as base (not clinical white, not stark). Soft tinted backgrounds. Content floats in whitespace rather than filling it.

**Anti-references** (explicitly NOT these):
- Generic SaaS dashboard with sidebar + card grid
- AI aesthetic: purple gradients, cyan accents, dark mode glow
- Jira / Linear density — this has only 3 features, not 300
- Terminal aesthetic — this is for everyone, not just CLI users
- Overwhelming card layouts — no card-in-card, no metric grids

**Typography**: Warm sans-serif paired with editorial serif for contrast. Hierarchy from size contrast, not weight proliferation. Skills and memories render as beautiful markdown — not raw text dumps.

**Color**: Warm, muted palette. One accent color (not blue, not purple). Tinted neutrals — even backgrounds have a subtle warmth. State colors are rare and meaningful.

**Layout**: Vertical flow with clear section breaks. Two-column max. Extreme whitespace discipline. No borders or dividers where spacing alone suffices. Soft rounded corners. Content breathes.

**Motion**: Slow, gentle. Fade-ins, not snaps. Emotional pacing. The product feels like it's always been here quietly working.

### Design Principles

1. **Three things, beautifully.** The product has exactly 3 tabs: Memories, Skills, Workflows. That's it. Every design decision serves clarity across these three, not feature sprawl.

2. **Intelligence, not configuration.** On first open, the dashboard should already show something useful — patterns found from scanning existing logs/work. The user shouldn't have to set things up to see value. "Automatically things start getting better."

3. **Concise yet informed.** Dense information displayed without feeling overwhelming. A skill isn't a wall of text — it's a refined card with just enough to understand what it does and when it applies. Show depth on interaction, not on first glance.

4. **Cumulative, not transactional.** Every visit should show MORE than last time — new memories, refined skills, workflow runs. The product compounds. The design should communicate this accumulation subtly (counts, timelines, growth indicators) without metric-dashboard syndrome.

5. **No slop.** If a UI element isn't required, it doesn't exist. No decorative icons, no placeholder cards, no empty states that just say "nothing here." Every pixel earns its place.
